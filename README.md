# Telegram Bot для авто-постинга космических фото

> Это учебный проект, созданный в рамках обучения на курсе [Профессия Middle Python/Django разработчик](https://dvmn.org/) от [Devman (dvmn.org)](https://dvmn.org/).

Проект для автоматической публикации фотографий космоса (NASA, SpaceX) в Telegram-канал. Бот умеет как отправлять одиночные фото вручную, так и работать в режиме 24/7, перебирая изображения из указанной папки.

## Установка

1. Клонируйте репозиторий и перейдите в папку:
   ```bash
   cd api_lesson4_dvmn
   ```

2. Создайте виртуальное окружение и активируйте его (рекомендуется):
   ```bash
   python -m venv .venv
   source .venv/bin/activate  # Linux/Mac
   .venv\Scripts\activate     # Windows
   ```

3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

## Настройка

Создайте файл `.env` в корне проекта и добавьте туда следующие переменные:

```ini
NASA_API_KEY=ВАШ_КЛЮЧ_NASA
TELEGRAM_BOT_TOKEN=ВАШ_ТОКЕН_БОТА
TELEGRAM_CHANNEL_ID=ID_ВАШЕГО_КАНАЛА

# Настройки прокси (опционально, если нужно для скачивания фото или работы бота)
HTTP_PROXY=socks5://127.0.0.1:9050
HTTPS_PROXY=socks5://127.0.0.1:9050
```

*   **NASA_API_KEY**: Можно получить на [api.nasa.gov](https://api.nasa.gov/) или использовать `DEMO_KEY` (с ограничениями).
*   **TELEGRAM_BOT_TOKEN**: Токен, полученный у [@BotFather](https://t.me/BotFather).
*   **TELEGRAM_CHANNEL_ID**: ID канала (например, `-100123456789`). Бот должен быть добавлен в канал как администратор.
*   **HTTP_PROXY / HTTPS_PROXY**: Адреса прокси-серверов. Поддерживаются `socks5://`, `socks5h://`, `http://`. Если прокси не нужен — эти строки можно не добавлять. После добавления используйте флаг `-p` при запуске скриптов.

## Скачивание фотографий

Перед публикацией нужно скачать изображения. В проекте есть скрипты для загрузки фото:

*   **NASA APOD (Astronomy Picture of the Day):**
    ```bash
    python fetch_nasa_apod.py [-p]
    ```
*   **NASA EPIC (Earth Polychromatic Imaging Camera):**
    ```bash
    python fetch_nasa_epic.py [-p]
    ```
*   **SpaceX Launch Photos:**
    ```bash
    python fetch_spacex_images.py [-p]
    ```
*Флаг `-p` включает использование прокси (если настроены переменные `HTTP_PROXY`/`HTTPS_PROXY`).*

Все фото сохраняются в папку `images/`.

## Использование

### 1. Ручной постинг (`bot.py`)

Отправляет одно фото из папки `images`.

*   **Случайное фото:**
    ```bash
    python bot.py
    ```
*   **Конкретное фото:**
    ```bash
    python bot.py spacex_0.jpg
    # или с полным путем
    python bot.py images/spacex_0.jpg
    ```
*   **С прокси:**
    ```bash
    python bot.py -p
    ```

### 2. Авто-постинг (`auto_post.py`)

Запускает бесконечный цикл публикации. Бот перемешивает фото в папке и отправляет их по одному с заданным интервалом. Когда все фото закончатся — цикл начинается заново (с новым перемешиванием).

**Примеры запуска:**

*   **По умолчанию (каждые 4 часа):**
    ```bash
    python auto_post.py
    ```
*   **Каждые 2 часа (через аргумент):**
    ```bash
    python auto_post.py --interval 2
    ```
*   **Указать другую папку и прокси:**
    ```bash
    python auto_post.py my_photos --interval 0.5 -p
    ```

**Обработка ошибок:**
При временных сбоях сети (таймаут) скрипт подождет 60 секунд и попытается отправить фото повторно. При критических ошибках инициализации бот будет пытаться переподключиться каждые 60 секунд.
